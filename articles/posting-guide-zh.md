# 各平台发文宣传指南

本文档提供了在 DEV Community, Hacker News, Reddit, 和 Product Hunt 四个平台上发布宣传文章的具体步骤和注意事项，以帮助 "Tab Storage Copier" 获得最大程度的关注。

---

### 1. DEV Community (dev.to)

*   **平台特点**: 国际化的开发者社区，氛围友好，适合发布详细的技术故事和教程。
*   **发布步骤**:
    1.  访问 [dev.to](https://dev.to/) 并注册/登录。
    2.  点击右上角的 "Write a Post" 按钮。
    3.  **标题**: 使用 `articles/dev-community.md` 中建议的标题，例如 "I Was Tired of Manually Copying localStorage, So I Built a Browser Extension to Do It in One Click"。
    4.  **内容**: 将 `articles/dev-community.md` 的全部内容复制粘贴到编辑器中。
    5.  **标签 (Tags)**: 添加相关的标签是获得曝光的关键。建议使用：`#javascript`, `#webdev`, `#productivity`, `#beginners`。
    6.  **封面图 (Cover Image)**: 上传一张吸引人的图片，例如 `pictures/promo_large.png`。
    7.  点击 "Publish" 发布。

---

### 2. Hacker News (news.ycombinator.com)

*   **平台特点**: 极客和技术创业者社区，内容简洁、直奔主题。成功会带来巨大流量。
*   **发布步骤**:
    1.  访问 [news.ycombinator.com](https://news.ycombinator.com/) 并注册/登录（需要一定的 "karma" 才能发帖，如果账号是新的，可能需要先参与一些讨论）。
    2.  点击顶部的 "submit"。
    3.  **标题 (title)**: 必须以 `Show HN:` 开头。复制 `articles/hacker-news.md` 中的标题：`Show HN: Tab Storage Copier – A browser extension to copy storage between tabs`。
    4.  **URL**: 填写你的 Chrome 商店链接或项目主页。
    5.  **文本 (text)**: **留空！** "Show HN" 类型的帖子不允许同时有 URL 和 text。
    6.  点击 "submit"。
    7.  **关键一步**: 提交后，立刻回到你的帖子，**在评论区发表第一条评论**。将 `articles/hacker-news.md` 中准备好的 "First Comment" 内容粘贴进去，对项目进行补充说明。

---

### 3. Reddit

*   **平台特点**: 由众多名为 "subreddit" 的子社区组成，每个社区都有自己的文化和规则。
*   **发布步骤**:
    1.  选择合适的 subreddit。推荐顺序：
        *   `r/SideProject`: 最适合分享个人项目，氛围最友好。
        *   `r/webdev`: 专业的 Web 开发者社区。
        *   `r/javascript`: JS 开发者社区。
        *   `r/frontend`: 前端开发者社区。
    2.  **仔细阅读社区规则 (Rules)**，确保你的帖子没有违规（例如，禁止过度推广）。
    3.  点击 "Create Post"。
    4.  **标题**: 使用 `articles/reddit.md` 中建议的标题。
    5.  **内容**: 将 `articles/reddit.md` 的内容复制粘贴进去。Reddit 支持 Markdown，图片链接会自动显示为图片。
    6.  发布后，花时间在评论区与用户互动，回答他们的问题。真诚的互动是在 Reddit 上获得成功的关键。

---

### 4. Product Hunt (producthunt.com)

*   **平台特点**: 全球最大的新产品发现社区。一次成功的发布可以带来媒体报道和大量早期用户。**发布需要提前准备**。
*   **发布步骤 (建议提前一天准备)**:
    1.  **寻找 "Hunter" (可选但推荐)**: 在 Product Hunt 上有影响力的用户 (Hunter) 可以帮助你的产品获得更多初始曝光。你可以在 Twitter 上寻找他们并向他们推荐你的产品。如果找不到，自己发布也可以。
    2.  访问 [www.producthunt.com/post/new](https://www.producthunt.com/post/new) 并开始提交。
    3.  **产品链接**: 填写项目主页或 Chrome 商店链接。
    4.  **产品名称 (Name of the product)**: `Tab Storage Copier`。
    5.  **标语 (Tagline)**: 复制 `articles/product-hunt.md` 中的标语。
    6.  **缩略图 (Thumbnail)**: 强烈建议使用一个展示核心功能的 GIF 动图。
    7.  **相册 (Gallery)**: 上传所有准备好的截图。
    8.  **主题 (Topics)**: 选择相关主题，如 `Developer Tools`, `Productivity`, `Chrome Extensions`。
    9.  **描述 (Description)**: 复制 `articles/product-hunt.md` 中的描述。
    10. **制作者评论 (First Comment)**: 在提交过程中，会有地方让你填写 "Maker's Comment"。将 `articles/product-hunt.md` 中准备好的第一条评论内容粘贴进去。
    11. **发布时间**: 选择一个合适的时间发布。通常建议在美国东部时间午夜（即太平洋时间晚上9点）发布，这样你的产品可以在整个美国工作日内都保持可见。
    12. **发布后**: 全天保持在线，积极回复评论区的每一个问题和反馈。
